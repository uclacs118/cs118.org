## Overview

In this project, you will implement a **TLS-like secure transport protocol** in C. You will build a secure client and server that:

1. Perform a **TLS handshake** using ECDH key exchange and X.509 certificates
2. Authenticate both sides using HMAC-SHA256
3. Encrypt data in transit using **AES-256-CBC**

This project gives practical experience with:

- ECDH (Elliptic Curve Diffie-Hellman) key exchange
- X.509 certificate parsing and verification
- AES-256-CBC encryption
- HMAC-SHA256 authentication
- TLV (Type-Length-Value) protocol encoding

## Protocol Details

### Handshake Flow

```
Client                          Server
  |                               |
  |-------- Client Hello -------->|
  |   (nonce + public key)        |
  |<------ Server Hello ----------|
  |   (nonce + certificate       |
  |    + signature)               |
  |-------- Finished ----------->|
  |   (HMAC transcript)           |
  |<------ Finished --------------|
  |   (HMAC transcript)           |
  |                               |
  |<========== Data ==============>|
  |   (AES-256-CBC + HMAC)       |
```

### Security States

| State | Role | Description |
| --- | --- | --- |
| `CLIENT_HELLO_SEND` | Client | Awaiting send of Client Hello |
| `CLIENT_SERVER_HELLO_AWAIT` | Client | Awaiting Server Hello |
| `CLIENT_FINISHED_SEND` | Client | Awaiting send of Finished |
| `SERVER_CLIENT_HELLO_AWAIT` | Server | Awaiting Client Hello |
| `SERVER_SERVER_HELLO_SEND` | Server | Awaiting send of Server Hello |
| `SERVER_FINISHED_AWAIT` | Server | Awaiting Finished |
| `DATA_STATE` | Both | In secure data exchange |

### Certificate Format (TLV)

The server certificate contains:

- `DNS_NAME` (0xA1): The server's DNS name (e.g. `localhost`)
- `PUBLIC_KEY` (0x02): Server's ephemeral EC public key
- `SIGNATURE` (0xA2): CA signature over DNS name + public key + lifetime
- `LIFETIME` (0xA3): Validity window (not_before, not_after as 64-bit big-endian)

### Key Cryptographic Parameters

- **Key Exchange**: ECDH with P-256 curve
- **Encryption**: AES-256-CBC
- **MAC**: HMAC-SHA256 (32 bytes)
- **Nonce**: 32 bytes (random)
- **IV**: 16 bytes (random per message)
