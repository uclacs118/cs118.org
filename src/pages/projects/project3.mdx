## Overview

In this project, you will implement a **TLS-like secure transport protocol** in C. You will build a secure client and server that:

1. Perform a **TLS handshake** using ECDH key exchange and X.509 certificates
2. Authenticate both sides using HMAC-SHA256
3. Encrypt data in transit using **AES-256-CBC**

This project gives practical experience with:

- ECDH (Elliptic Curve Diffie-Hellman) key exchange on the P-256 curve
- X.509 certificate parsing and verification
- AES-256-CBC encryption and decryption
- HMAC-SHA256 message authentication
- TLV (Type-Length-Value) binary protocol encoding

## Protocol Details

### Handshake Flow

```
 Client                                        Server
   |                                              |
   |  1. CLIENT HELLO  (nonce + client pubkey)    |
   |  ──────────────────────────────────────────> |
   |                                              |
   |  2. SERVER HELLO  (nonce + certificate)      |
   |      + SERVER SIGNATURE                      |
   |  <─────────────────────────────────────────  |
   |                                              |
   |  3. CLIENT FINISHED  (HMAC of transcript)    |
   |  ──────────────────────────────────────────> |
   |                                              |
   |  4. SERVER FINISHED  (HMAC of transcript)   |
   |  <─────────────────────────────────────────  |
   |                                              |
   |  5. ENCRYPTED DATA  (AES-256-CBC + HMAC)    |
   |  ════════════════════════════════════════════ |
   |  (both directions, until connection closes)  |
```

### Security States

| State | Role | Meaning |
|---|---|---|
| `CLIENT_CLIENT_HELLO_SEND` | Client | About to send Client Hello |
| `CLIENT_SERVER_HELLO_AWAIT` | Client | Waiting for Server Hello + Certificate |
| `CLIENT_FINISHED_SEND` | Client | About to send Client Finished |
| `SERVER_CLIENT_HELLO_AWAIT` | Server | Waiting for Client Hello |
| `SERVER_SERVER_HELLO_SEND` | Server | About to send Server Hello + Certificate |
| `SERVER_FINISHED_AWAIT` | Server | Waiting for Client Finished |
| `DATA_STATE` | Both | Handshake done — in secure data exchange |

### TLV (Type-Length-Value) Encoding

All messages are encoded as TLV structures. The format:

```
  1 byte     1 or 3 bytes           N bytes
+--------+------------------+---------------------+
|  Type  |  Length (LE)     |  Value              |
+--------+------------------+---------------------+
```

- **Type**: 1 byte identifying the field
- **Length**: 1 byte if value length ≤ 252 bytes, else 3 bytes (`0xFD` + 2-byte big-endian length). The threshold is `VN3 - 1 = 0xFC` (decimal 252).
- **Value**: raw bytes (primitive) or nested TLVs (container)

**TLV Container types** (their value is a sequence of child TLVs):

| Type | Name | Description |
|---|---|---|
| `0x10` | `CLIENT_HELLO` | Client's opening message |
| `0x20` | `SERVER_HELLO` | Server's response message |
| `0xA0` | `CERTIFICATE` | X.509 certificate (contains children) |
| `0x30` | `FINISHED` | Handshake completion message |
| `0x50` | `DATA` | Application data record |

**Primitive types** (their value is raw bytes):

| Type | Name | Value Size | Description |
|---|---|---|---|
| `0x01` | `NONCE` | 32 bytes | Random nonce for ECDH |
| `0x02` | `PUBLIC_KEY` | 65 bytes | Uncompressed EC point (P-256) |
| `0xA1` | `DNS_NAME` | variable | Server DNS hostname (e.g. `localhost`) |
| `0xA2` | `SIGNATURE` | 64 bytes | ECDSA signature over transcript |
| `0xA3` | `LIFETIME` | 16 bytes | `not_before` + `not_after` (64-bit big-endian each) |
| `0x51` | `IV` | 16 bytes | Random IV for AES-CBC |
| `0x52` | `CIPHERTEXT` | variable | AES-256-CBC encrypted data |
| `0x53` | `MAC` | 32 bytes | HMAC-SHA256 tag |
| `0x04` | `TRANSCRIPT` | variable | HMAC of all handshake messages so far |

### Certificate Contents

The `CERTIFICATE` TLV (`0xA0`) contains four child TLVs:

```
CERTIFICATE (0xA0)
  ├── DNS_NAME      (0xA1)  : "localhost"
  ├── PUBLIC_KEY    (0x02)  : server's ephemeral EC public key
  ├── SIGNATURE     (0xA2)  : CA's ECDSA signature over DNS_NAME + PUBLIC_KEY + LIFETIME
  └── LIFETIME      (0xA3)  : 8 bytes not_before + 8 bytes not_after (big-endian)
```

The CA's public key is in `keys/ca_public_key.bin` (65-byte uncompressed EC point).

### Cryptographic Parameters

| Parameter | Value |
|---|---|
| Key exchange | ECDH on P-256 curve (`NID_X9_62_prime256v1`) |
| Signature | ECDSA with P-256 (64-byte r||s encoding) |
| Encryption | AES-256-CBC |
| MAC | HMAC-SHA256 (32 bytes) |
| Client nonce | 32 random bytes |
| Server nonce | 32 random bytes |
| AES IV | 16 random bytes per encrypted record |

### File Locations

| File | Location | Purpose |
|---|---|---|
| CA public key | `keys/ca_public_key.bin` | Verify server certificate signature |
| Server private key | `keys/server_key.bin` | Sign server's nonce+pubkey in `handle_client_hello` |
| Server cert (valid) | `keys/server_cert.bin` | Sent to client during handshake |
| Server cert (expired) | `keys/server_cert_expired.bin` | Used by expired-cert test only |

### TLV Encoding Reference

The provided framework handles TLV serialization/deserialization. Key functions to use:

```c
// Read a TLV from a buffer
tlv* deserialize_tlv(uint8_t* buf, uint16_t size);

// Write a TLV to a buffer
uint16_t serialize_tlv(uint8_t* buf, tlv* t);

// Find a child TLV by type inside a container
tlv* get_tlv(tlv* parent, uint8_t type);
```

Primitive fields: `NONCE` (0x01), `PUBLIC_KEY` (0x02), `DNS_NAME` (0xA1), `SIGNATURE` (0xA2), `LIFETIME` (0xA3), `IV` (0x51), `CIPHERTEXT` (0x52), `MAC` (0x53), `TRANSCRIPT` (0x04).

Container fields (use `get_tlv` to find children): `CLIENT_HELLO` (0x10), `SERVER_HELLO` (0x20), `CERTIFICATE` (0xA0), `FINISHED` (0x30), `DATA` (0x50).

### Implementation Hints

1. **Start with `handle_client_hello`** — it's called first when the client connects. Build a `CLIENT_HELLO` TLV, send it.
2. **Use `get_tlv`** to extract the server's nonce and public key from its `SERVER_HELLO` message.
3. **For ECDH**: Use `EVP_PKEY_new_raw_public_key` with `EVP_PKEY_X25519` or the P-256 curve to import the raw 65-byte EC point. Call `EVP_PKEY_derive` to compute the shared secret.
4. **For the transcript HMAC**: Collect every raw bytes of every handshake message (including type+length headers) in order. HMAC-SHA256 the concatenated transcript.
5. **The expired cert test**: Your `handle_server_hello` should call `ntohl()` on the 64-bit big-endian timestamps in `LIFETIME`, compare to `time(NULL)`, and return without sending if expired.
6. **AES-CBC**: Use `EVP_CIPHER_CTX_new`, `EVP_EncryptInit_ex2`, `EVP_CipherUpdate`, `EVP_CipherFinal_ex`. The IV is sent in the clear before the ciphertext.
